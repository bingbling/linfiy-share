## 分支

方便的分支操作才是！而且我们早就在用啦！

> 分支赋予我们穿行于多维空间的能力。 

### 场景1

高考的数学考场，最后一道数学证明题。证明EFGH是平行四边形。

1. 思路一：证明两组对边分别平行
2. 思路二：证明两组对边分别相等
3. 思路三：证明一组对边平行且相等
4. 思路四：证明两组对角相等的四边形
5. 思路五：对角线互相平分

你不大可能有思路一的时候便在试卷上比划起来。没错，此时草纸便成为了你的分支。当你过滤剩下了一两个方法之后你会把草纸分支上的代码合并到试卷上(主分支)


### 场景2

由于高考数学考爆炸了，你无奈学了设计，大学毕业后你每天忙于做海报。下周是建军节，老板让你用儿童节的海报改6个稿让他决定用哪张做宣传。

你不大可能复制出6张儿童节的海报，然后在第一张上加一架飞机表白空军，在第二张海报上加一艘航母表白海军... 极有可能的你会新建一些图层加上飞机航母坦克，然后导出不同的组合。

此时图层就是分支。最后导出时合并了所有可见的图层，完成老板想要的海报。

### 场景3

由于你忘记把海报里儿童的脸拉长了，因此暴露了公司造假的行为。因此建军节的棉花糖销售策略完全失败！你不得不回家待业。此时你找到了一个网络写手的工作。主编的要求不高，每天更新十章即可。这对你来说简直是小菜一碟。

在你写作正high的时候，主编要你把前两千章交给他拍成纪录片。你复制出了一个文件夹把两千章放了进去。主编很满意，并给出了你一些小小的建议，章节的标题从2级改为3级


## 简单命令

`暂存 staged`

### `git status` 查看版本库的当前状态

### `git init` 创建版本库

创建唯一一个`.git` 隐藏文件夹在项目的顶层目录

回退: `rm -rf .git`

### `git add <fileName>` 将文件添加到版本库中

回退：
  - `git reset HEAD <fileName>`
  - `git rm --cached <fileName>`

添加许多文件 `git add .` / `git add *`

回退：
  - `git reset HEAD .`
  - `git rm -r --cached .` -r 允许递归删除
  - `git reset --mixed`


## `git commit` 将暂存区的代码提交到版本库中

每次提交都需要输入提交的信息，`git commit -m "message"`, 如果你需要输入更为详细的信息，则直接输入 `git commit` 会唤醒 `vi`

### 如果想要唤醒你常用的编辑器进行编辑需要进行配置

- Windows

  1. 首先将你的编辑器添加到环境变量中
  eg: `我的电脑` - `右键属性` - `左侧高级系统设置` - `环境变量` - `PATH` 中添加 `D:\software\work\Microsoft VS Code\bin`
  
  2. 命令行执行 `git config core.editor code[sublime][atom]...`

- OSX
  1. 添加环境变量
  2. 命令行执行 `git config core.editor code[sublime][atom]...`


没有改变文件关闭编辑模式，则会取消这次提交。
